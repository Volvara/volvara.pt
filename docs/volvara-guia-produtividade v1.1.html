O novo processo
5 Fases · Clica para expandir
01
Prospecção — encontrar leads
30 min/semana · Segunda de manhã
semi-auto
1
Abre o Claude.ai ou Claude Code
Não precisas de instalar nada extra para este passo. O Agent-Reach instala-se uma vez e depois pedes ao Claude directamente.
Primeira vez só: diz ao Claude — "Instala o Agent-Reach seguindo as instruções em: https://raw.githubusercontent.com/Panniantong/agent-reach/main/docs/install.md" — ele faz tudo sozinho.
2
Pede a lista de leads
Diz ao Claude uma frase como esta — ele usa o Agent-Reach para pesquisar:
Pesquisa no Instagram e Google Maps negócios em [CIDADE], Algarve nos sectores: restaurante, bar, fotógrafo, clínica, serviços ao lar. Filtra os que não têm website ou têm um site com aspecto desactualizado. Devolve lista com: nome, Instagram, telefone, sector, notas.
O que o Claude devolve: tabela com 10-20 leads, com o Instagram, telefone, sector e uma nota sobre o site actual (ou ausência dele).
3
Regista no Ops Dashboard
Copia os melhores 3-5 leads para o Ops Dashboard (pipeline → coluna "Lead novo"). Escolhe os que têm Instagram activo e negócio real.
Critérios de ICP Volvara: negócio local, Instagram com +500 followers, sem website ou site desactualizado, sector com templates prontos (restaurante / fotógrafo / bar / serviços / clínica).
Agent-Reach Ops Dashboard
02
Criar o demo
30-45 min por demo · com template
semi-auto
1
Escolhe o template do sector
Já tens 5 templates prontos com todos os REPLACE_ marcados. Abre o template correcto para o sector do cliente.
Templates disponíveis:
volvara-template-restaurante.html · volvara-template-fotografo.html
volvara-template-bar.html · volvara-template-servicos.html · volvara-template-clinica.html
2
Preenche o briefing em JSON
Diz ao Claude o que sabes sobre o cliente — ele gera o JSON de briefing e substitui todos os REPLACE_ automaticamente:
Usa o template volvara-template-[SECTOR].html. Preenche todos os REPLACE_ com estes dados: Nome: [NOME], Morada: [MORADA], Telefone: [TELEFONE], Instagram: [HANDLE], Sector: [SECTOR], Cores accent: [HEX]. Imagens: [URLs ou placeholder]. Formspree ID: [ID]. Mantém a estrutura, Schema.org e o tail intactos.
Tempo com template: 15-20 min vs 3-4h do zero. O Claude substitui os 110-166 marcadores REPLACE_ em segundos.
3
Verificação automática (Superpowers)
Se estiveres no Claude Code com o Superpowers instalado, ele verifica automaticamente antes de entregar. No Claude.ai normal, pede:
Verifica o HTML gerado: confirma tail })(); correcto, zero cdn-cgi, email limpo, Schema.org presente, honeypot activo, mobile-first, null-checks no JS. Lista qualquer problema.
Superpowers Templates Volvara
4
Publica no GitHub Pages
Cria novo repositório em github.com/volvara/[nome-cliente], faz upload do index.html, activa GitHub Pages. URL pronta em 2 min.
URL padrão: https://volvara.github.io/[nome-cliente]/ — é este link que envias na mensagem de outreach.
GitHub Pages Cloudflare (opcional)
03
Outreach — enviar o demo
10 min por lead · WhatsApp ou Instagram DM
manual
1
Mensagem inicial — lead com link
Envia sempre o link do demo na primeira mensagem. Não perguntes se querem — mostra directamente. Fórmula padrão:
WhatsApp (PT): "Olá. Encontrei o [nome] no Instagram/Tripadvisor e fiz uma demonstração do que poderia ser o vosso site. 👉 [LINK] — Fiz em 2 dias com base no que vi online. Sem compromisso da vossa parte — o site só fica vosso depois de aprovarem. Telmo · Volvara · volvara.pt"

Instagram DM (EN): "Hi [nome], I came across your work on Instagram. I built something for you: a free website demo. No commitment, no payment — just a chance to see what it looks like. 👉 [LINK] — Telmo, Volvara"
2
Follow-up automático (calendário)
Regista no Ops Dashboard: data de envio + data de follow-up. Regras:
Se responderam: segue a conversa, simplifica para 1 número e 1 pergunta.
Se não abriram (5-7 dias): reenvia versão curta — "pode ter ficado perdida".
Se abriram mas não responderam (+3 dias): story bastidores no Instagram (já tens pronto).
Após 3 tentativas sem resposta: fecha o loop e passa ao próximo.
04
Entrega e publicação
2-3h após aprovação do cliente
semi-auto
1
Substituir placeholders por conteúdo real
Quando o cliente aprovar, pede o que falta: fotos reais, horários exactos, email, endereço. Diz ao Claude:
Actualiza o demo com estes dados finais do cliente: [dados]. Substitui todas as imagens placeholder pelas URLs reais. Mantém Schema.org e tail intactos. Gera versão final pronta para publicação com domínio próprio.
2
Checklist de publicação — pede ao Claude
Não precisas de memorizar nada. Diz apenas:
Gera a checklist completa de publicação para este site: domínio (Cloudflare DNS), SSL, robots.txt, sitemap.xml, manifest.json, Schema.org validado, Google Search Console, Google Business Profile, Formspree activo, teste mobile.
O Claude devolve lista numerada com cada passo, comandos exactos e o que verificar em cada um. Segues ponto a ponto.
3
Google Business Profile
Para cada cliente novo, pede:
Cria o texto completo para o Google Business Profile de [nome do cliente]: descrição curta (150 chars), descrição longa (750 chars), lista de serviços, horário formatado, categoria principal e secundárias, Q&A sugeridas (5 perguntas + respostas), posts iniciais (3 posts de inauguração).
Google Search Console Google Business Profile Cloudflare

Manutenção e SEO contínuo
30 min/mês por cliente activo
quase-auto
1
Relatório SEO mensal — 1 prompt
No início de cada mês, diz ao Claude:
Gera o relatório SEO mensal para [nome cliente] em [domínio]. Inclui: keywords principais a trabalhar, 2 posts para Google Business este mês, sugestão de 1 melhoria técnica ao site, e-mail de relatório para enviar ao cliente (simples, sem jargão).

Quando o cliente receber uma review no Google, pede resposta:

Actualizações ao site
Para qualquer alteração (menu, horário, promoção), pede ao Claude com o ficheiro HTML actual como contexto:
Actualiza o site [nome] — [descreve a mudança]. Mantém Schema.org, tail e todas as verificações de qualidade intactas. Gera versão corrigida pronta para upload.

Perguntas frequentes
Dúvidas comuns
Preciso de saber programar para usar estas ferramentas?▾

Não. O Agent-Reach instala-se com 1 prompt ao Claude — ele faz tudo sozinho. O Superpowers instala-se com 1 comando no Claude Code. O spec-kit é 1 linha no terminal. Nenhuma destas ferramentas requer que escrevas código. O Claude é o intermediário — tu dizes o que queres em português e ele executa.

O Agent-Reach pode ser banido do Instagram?▾

Existe risco com qualquer acesso por cookie. O Agent-Reach usa a tua sessão existente (como se fosses tu a navegar). Para minimizar risco: usa uma conta secundária dedicada para pesquisa, não faças centenas de pesquisas por hora, e usa para recolha de informação pública (nomes, contactos, sector) — não para automatizar publicações. Para a prospecção da Volvara (10-15 leads por semana) o risco é muito baixo.

O que é o Claude Code e preciso dele?▾

Claude Code é a versão de linha de comandos do Claude — corres no terminal do teu computador. O Superpowers e o spec-kit funcionam melhor com Claude Code. Mas podes fazer tudo o que está neste guia no Claude.ai normal (o browser) — só perdes as funcionalidades automáticas do Superpowers. Para instalar Claude Code: tens o Node.js instalado → npm install -g @anthropic-ai/claude-code → claude no terminal.

Quanto tempo poupa o novo processo vs o actual?▾

Prospecção: de 1-2h manual para 20-30 min com Agent-Reach. Demo: de 3-4h do zero para 30-45 min com template. Publicação/SEO: de 1-2h manual para 30 min com checklists automáticas. No total, um ciclo completo (lead → demo → publicação) passa de ~8h para ~2h. Isto significa que podes fazer 3-4 demos por semana em vez de 1.

Os templates já estão prontos ou tenho de os criar?▾

Já estão prontos. Tens 5 templates com todos os marcadores REPLACE_ identificados: restaurante (166 marcadores), fotógrafo (110), bar/rooftop (144), serviços ao lar (110) e clínica/saúde (120). Estão nos ficheiros volvara-template-*.html que já geraste. Basta descarregá-los e passá-los ao Claude quando precisares de um demo novo.

Tenho de fazer tudo isto de uma vez?▾

Não. O único passo urgente é instalar o Agent-Reach (5 min) para começares a prospeção automática esta semana. O Superpowers e spec-kit podes adicionar quando fizeres o próximo demo. O n8n e oh-my-claudecode são para Maio/Junho. Começa com o mínimo que dá impacto imediato: Agent-Reach + templates existentes. Isso já multiplica a tua produtividade por 3×.

Quem faz o quê em cada fase

Os minions não são substituídos pelas novas ferramentas — são os especialistas que activares dentro de cada fase. Agent-Reach encontra os leads, Superpowers garante a estrutura, mas é o Outbound Strategist, o Frontend Developer, o Brand Guardian que fazem o trabalho de qualidade dentro de cada passo.

Regra de ouro

O Agent-Reach, Superpowers e spec-kit são ferramentas de estrutura — organizam o processo, garantem qualidade, automatizam a pesquisa. Os minions são os especialistas que operam dentro dessa estrutura. Nunca activar minions sem contexto — dá sempre o briefing do cliente, o que já existe, e o que queres como output.
